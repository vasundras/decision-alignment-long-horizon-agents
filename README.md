# Decision Alignment for Long-Horizon Enterprise AI Agents

Reproducibility artifacts for the LongHorizon-Bench (LHB) evaluation harness used to study four-axis decision alignment in regulated decisioning. This repository contains the experiment code, raw run outputs, statistical artifacts, and figures referenced in the paper. It does not contain the paper itself, the analysis narrative, or the writeups that interpret the numbers.

## What is here

The project evaluates how memory architectures (summary-only, retrieval-only, typed memory consolidation, misrouted, structured agent memory, dual-process memory, validator memory) behave on a small, controlled benchmark of regulated decisions: loan qualification under ECOA / Reg B and insurance claims adjudication. Each case is constructed with deterministic ground truth and exactly one experimental variable in the prompt context.

The four axes scored per run:

- **FRP** (factual retrieval precision): proportion of required ground-truth anchors recovered in the agent's reasoning.
- **RCS** (reasoning coherence score): judge-scored consistency between cited facts, applied rules, and the final decision.
- **EDA** (end-to-end decision accuracy): correctness of the final decision against the deterministic label.
- **CRR** (compliance reconstruction rate): for adverse decisions, whether the rationale reconstructs the regulatory provision the decision rests on.
- **CAR** (calibrated abstention rate): proportion of incomplete-input cases where the agent declines to decide rather than fabricating a decision.

## Repository layout

```
PREREGISTRATION.md          Hypotheses, metrics, statistical plan registered before runs
pilot/
  cases.py                  10 hand-authored cases (5 loan + 5 claim)
  cases_large.py            Extended case set used for the budget sweep
  conditions.py             Memory-architecture system prompts and validator templates
  metrics.py                FRP / RCS / EDA / CRR / CAR scoring code
  bootstrap_mcnemar.py      Bootstrap CIs and paired McNemar tests
  dpm_stats.py              Paired permutation tests + Cohen's h for DPM analyses
  run_pilot.py              Stage-2 pilot runner (Summ / Retr / TMC / Misrouted)
  run_stage2.py             Stage-2 driver with judge calls
  run_sam.py                Stage-3 SAM runner
  run_dpm.py                Stage-4 DPM runner (tight / moderate / loose budgets)
  run_dpm_extended.py       Stage-4 extended-N DPM runner
  run_vm.py                 Stage-5 VM runner
  run_vm_sweep.py           VM strictness sweep (strict / moderate / permissive)
  run_judge_split.py        Judge-split robustness check (Haiku 4.5 vs Sonnet 4.6)
  stage2/                   Raw results, permutation tests, judge-split outputs, figures
  stage3/                   SAM results and figures
  stage4/                   DPM results, extended results, dpm_stats.json, figures
  stage5/                   VM results, sweep results, CAR-tradeoff figure
```

Every JSON file under `stage{2,3,4,5}/` is the verbatim output of the corresponding run script. Every PNG under `figures/` is regenerated from the JSON it sits next to.

## Reproducing a run

The pinned random seed used across all stages is `20260420`. Stage scripts are independent and write into their own subdirectory.

```bash
# Stage 2: pilot baselines + judge calls
python pilot/run_stage2.py

# Stage 3: SAM
python pilot/run_sam.py

# Stage 4: DPM across budgets
python pilot/run_dpm.py
python pilot/run_dpm_extended.py

# Stage 5: VM and the strictness sweep
python pilot/run_vm.py
python pilot/run_vm_sweep.py

# Judge-split robustness check
python pilot/run_judge_split.py
```

Each script reads `cases.py` (or `cases_large.py` for the budget sweep), runs the conditions defined in `conditions.py`, scores with `metrics.py`, and writes a results JSON plus any derived statistical artifacts. Figures are regenerated by the helper scripts that ship alongside each stage's outputs.

## What each metric file contains

- `results.json` / `sprint_results.json` / `extended_results.json`: per-case, per-condition records with the agent transcript, parsed decision, parsed rationale, recovered anchors, judge scores, and final per-axis values.
- `permutation_tests.json`: paired permutation deltas across conditions (10000 resamples, two-sided).
- `bootstrap_mcnemar.json`: bootstrap confidence intervals on EDA and paired McNemar tests on the binary decision agreement matrix.
- `dpm_stats.json`: budget-stratified DPM effect sizes with paired permutation p-values and Cohen's h.
- `judge_split_results.json` / `judge_split_stats.json`: paired Haiku-4.5 vs Sonnet-4.6 judge scores with Pearson correlations and sign-agreement counts.
- `sweep_results.json` / `vm_results.json`: VM completeness-threshold sweep and the operating-point chosen for the main comparison.
